## 这是什么

一份完整的 Agent 工程框架文档，整合了当前 Agent 工程领域四个最重要的知识来源：

| 来源 | 贡献 |
|------|------|
| [Karpathy LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) | 知识复利模式 — LLM 是维基维护者，不是检索工具 |
| [claude-obsidian](https://github.com/AgriciDaniel/claude-obsidian) | Karpathy 模式的完整工程化实现（15 skills, hot cache, hybrid retrieval） |
| [ADPS 模式矩阵](https://adpsagent.com/zh/patterns/) | 33 个 Agent 设计模式的双轴矩阵（7 认知功能 × 6 执行拓扑） |
| [agent-design-patterns](https://github.com/huangjia2019/agent-design-patterns) | ADPS 28 模式 LangGraph + LangChain 双框架可运行代码（作者：黄佳） |
| 企业级 Agent 工程治理 | 8 层治理栈 + FDE 交付机制 + AI 应用开发 SOP |

## 文档结构

7 个部分，从「为什么」到「怎么做」到「往哪走」：

1. **三个第一性原理** — 知识复利、约束外置、降低下次部署成本
2. **Karpathy LLM Wiki 模式** — 三层架构 + 三个操作 + claude-obsidian 工程化实现
3. **ADPS 33 模式** — 7×6 双轴矩阵 + 五个工程契约 + 六步选型法
4. **8 层工程治理栈** — 从工具接入到输出校验的完整约束链
5. **FDE 交付机制** — 从现场取证到产品反哺的 8 步循环
6. **技术趋势全景** — 10 个趋势 + 8 个机制的完整链路图
7. **AI 应用开发 SOP** — 10 步从场景定义到周复盘
